## 第一步：把网站部署到公网 HTTPS

### 方案A：Vercel（推荐，最简单）

Vercel 免费托管，自带 HTTPS，全球 CDN 加速。

**1. 注册账号**
打开 https://vercel.com ，点 Sign Up → 用 GitHub 账号登录（没有的话先注册一个 GitHub）

**2. 安装 Vercel CLI 并部署**
```bash
# 在你的 WSL 终端执行：
cd ~/tomato-matrix

# 安装 Vercel CLI
npm i -g vercel

# 登录（会打开浏览器，授权 GitHub 账号）
vercel login

# 部署！一路回车用默认选项
vercel --prod

# 部署成功后会得到一个 URL，比如：
# https://tomato-matrix-xxxxx.vercel.app
```

**3. 绑定自己的域名**
```bash
# 先去阿里云/腾讯云 DNS 控制台添加 CNAME 记录
# 然后告诉 Vercel 你的域名：
vercel domains add tomato.你的域名.com
```

> ⚠️ 注意：微信 WebView **不接受 Vercel 默认域名** `*.vercel.app`，必须用自己的域名。

### 方案B：Cloudflare Pages（备选）

类似 Vercel，也是免费 HTTPS。

1. 注册 Cloudflare → Workers & Pages → Pages → Create
2. 连接 GitHub 仓库，上传 `dist/` 目录
3. 同样需要自己的域名（`*.pages.dev` 微信也不接受）

### 方案C：腾讯云静态网站托管（国内访问更快）

1. 开通腾讯云 COS + CDN
2. 上传 `dist/` 目录
3. 开启 HTTPS + 绑定域名
4. 需 ICP 备案（免费，约 2 周）

## 第二步：注册微信小程序

**1. 注册**
打开 https://mp.weixin.qq.com → 立即注册 → 小程序

- 类型选「个人」
- 填写身份证信息
- 填写小程序名称（如"番茄象限工作法"）
- **不需要认证**（个人主体免认证费）
- 审核通常 1 个工作日

**2. 下载开发者工具**
https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html

安装后扫码登录。

**3. 配置服务器域名**
登录小程序后台 → 开发管理 → 开发设置 → 服务器域名

在 **业务域名** 栏添加你的 HTTPS 域名（如 `tomato.你的域名.com`）

> ⚠️ 微信要求业务域名的网站在根目录放置一个校验文件。Vercel/Cloudflare 部署后我们会在第四步处理。

我已经帮你生成好了，就在 `~/tomato-matrix-miniapp/` 目录。

这个目录就是一个完整的小程序项目，里面只有一个页面，用 `<web-view>` 嵌入你的番茄象限网站。

3. 修改 `app.json` 中的小程序名称
4. 在开发者工具中预览 → 上传 → 提交审核

小程序代码我已经生成好了，请看下一节。

## 第四步：域名校验文件

微信规定，业务域名的网站根目录必须放一个校验文件（从微信后台下载）。

**Vercel 做法：**
1. 在微信小程序后台 → 开发设置 → 业务域名 → 下载校验文件（如 `abc123.txt`）
2. 把文件放到 `~/tomato-matrix/public/` 目录
3. 重新部署：`vercel --prod`

**腾讯云做法：**
直接把文件上传到 COS 桶根目录。

## 常见问题

**Q: 个人小程序能用 web-view 吗？**
A: 可以。个人主体小程序完全支持 `<web-view>` 组件。

**Q: 一定要域名吗？**
A: 是的。微信对 web-view 的域名要求非常严格，必须 HTTPS + ICP 备案（国内服务器）或 HTTPS（海外服务器）+ 加入业务域名白名单。没有域名做不了。

**Q: 小程序体验和网页有区别吗？**
A: 使用 web-view 方案，体验完全一致。所有功能（拖拽、番茄钟、日历、导出）都正常。

**Q: 能不能不要域名？**
A: 如果只在自己手机上用，可以不用小程序，直接在手机浏览器打开部署好的 HTTPS 网站，添加到主屏幕（PWA 体验），和 App 几乎一样。
